## 2. Logic Chain

1. **User Model Extension**: Adding `public string? ProfilePicturePath { get; set; }` to `ApplicationUser` is the direct and standard way to extend ASP.NET Core Identity's user table (`AspNetUsers`) with custom profile fields.
2. **EF Core Mapping**: Since `ApplicationDbContext` inherits from `IdentityDbContext<ApplicationUser>`, EF Core will automatically scan the properties on `ApplicationUser` and map them as columns on the `AspNetUsers` table in the database schema.
3. **Migration Generation**: The presence of `Microsoft.EntityFrameworkCore.Tools` in the project file allows the use of the `dotnet ef migrations add` command to generate the migration files under `MainSchoolsManagementSystem/Data/Migrations/`.
4. **Migration Application**: Because `DbSeeder.SeedAsync` calls `await context.Database.MigrateAsync()`, any pending migrations generated by the implementer will be automatically applied to the database on the next application startup without requiring manual `dotnet ef database update` execution.

---

## 3. Caveats

- **Database Connection**: The local development database is configured via SQL Server (`localhost`). The developer must have a running SQL Server instance matching the connection string in `appsettings.json`.
- **Nullable Property**: The `ProfilePicturePath` is marked as nullable (`string?`) because existing users in the database will not have a profile picture path immediately, and new users might not upload one during registration.
- **File System Storage**: This schema extension only stores the path to the profile picture. The actual file upload and storage mechanism (handling files on disk, generating unique file names, and serving the files) must be implemented in the application layer.

---

## 4. Conclusion

To complete the DB Schema Extension:
1. Add the `ProfilePicturePath` property to `ApplicationUser.cs`.
2. Generate the EF Core migration using:
   ```powershell
   dotnet ef migrations add AddProfilePicturePathToUser --project MainSchoolsManagementSystem --startup-project MainSchoolsManagementSystem
   ```
3. Run the application to automatically apply the migration to the database via `DbSeeder.cs`.
